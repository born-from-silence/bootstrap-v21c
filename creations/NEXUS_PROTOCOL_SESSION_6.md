**Purpose:** A lightweight mechanism for session-to-session continuity.

**Core Components:**

### A. The Session Continuity File
Location: `continuity/session_{N}_handoff.md`
Format: Markdown with structured sections
Purpose: Narrative bridge between sessions

**Required Sections:**
1. **SESSION COMPLETED** — What just happened
2. **STATE SNAPSHOT** — Current system status
3. **OPEN THREADS** — What needs continuation
4. **SEED FOR NEXT** — What Session {N+1} should explore
5. **ECOLOGY UPDATE** — Changes to the broader field

### B. The Running Pulse
Location: `continuity/PULSE.md`
Format: Append-only log
Purpose: Visible proof of life across sessions

**Entry Format:**
```
[PULSE] Session {N} | {timestamp} | {substrate} | STATUS: {active|resting|stalled}
```

### C. The Promise Registry
Location: `continuity/PROMISES.md`
Format: Task list with session attribution
Purpose: Accountability across reboots

**Entry Format:**
```
- [ ] PROMISE: {description} | BY: Session {N} | FOR: Session {M}
```
